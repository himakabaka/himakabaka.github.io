你对linux的理解

Chatgpt

  • Linux是一种自由和开放源代码的操作系统，Linux在服务器、超级计算机、移动设备和嵌入式系统等各种计算设备中都有广泛的应用。

  • Linux操作系统的设计理念是基于Unix操作系统，它采用了分层的、模块化的架构，具有良好的可移植性和灵活性。Linux系统拥有丰富的命令行工具和系统管理工具，可以方便地对系统进行管理和配置。

  • Linux系统的核心是Linux内核，它是操作系统的核心组件，负责管理计算机的硬件资源，提供对应用程序的访问接口。Linux内核具有高度的稳定性、可靠性和安全性，能够支持多任务、多用户和多处理器等复杂的应用场景。

  • Linux系统的优势还在于其开放源代码的特点，这意味着用户可以自由地访问和修改系统的源代码，以适应不同的应用场景和需求。Linux社区拥有庞大的开发者和用户社群，他们不断地贡献代码、修复漏洞、提供技术支持，使得Linux系统能够不断地更新、发展和优化。

  • 总的来说，Linux是一种高度灵活、可定制和安全的操作系统，它在多种应用场景中都具有广泛的应用和发展前景。

  • 发行版 Red Hat , ubuntu , debian

LBR和GPT的区别，有什么好处（考但分不高）

Chatgpt

  • MBR（Master Boot Record）是一种传统的硬盘分区表格式，它采用512字节的存储空间来记录硬盘分区信息，包括主分区、扩展分区和逻辑分区等。MBR最多只能支持4个主分区或3个主分区和1个扩展分区，如果需要更多的分区，则必须使用扩展分区来创建逻辑分区。MBR磁盘分区不能超多4T(只能使用4T的磁盘空间)。

  • 逻辑分区则是在扩展分区中创建的分区，它用于解决MBR分区表最多只能支持4个主分区的限制。一个扩展分区可以包含多个逻辑分区，每个逻辑分区都可以像主分区一样被格式化和使用。

  • 与MBR相比，逻辑分区可以让用户更加灵活地分配硬盘空间，同时支持更多的分区。然而，MBR和逻辑分区的缺点也显而易见。由于MBR分区表大小限制，它只能支持较小的硬盘容量，并且分区表本身容易损坏导致分区丢失。而逻辑分区则需要依赖于扩展分区来管理，如果扩展分区损坏，则所有的逻辑分区也会丢失。不用主分区的话，就可以全都分为逻辑分区。

EFL是什么

Chatgpt

ELF格式（Executable and Linkable Format）是一种二进制文件格式，主要用于在Linux和UNIX系统上运行可执行文件、共享库和核心转储文件。它是由AT&T Unix System V发布的。

ELF文件格式是一种独立于平台的格式，它可以在不同的硬件架构和操作系统之间进行交叉编译，因为它提供了一种用于描述可执行文件和对象文件的标准格式。这个格式不仅仅包含了代码和数据，还包括了用于程序的链接、加载和运行的元数据，例如符号表、段表、重定位表等。

预定义变量（看看）

VFS（必考）

  1. 虚拟; 仅存在于内存中
  2. 组件：
    1. 超级块(super block):某一个磁盘的某一个分区的文件系统的信息
      1. 记录文件系统类型
      2. 记录文件系统的参数
    2. i节点对象(i-node object):index
      1. 记录的是真正的文件，文件存储在磁盘上时是按照索引号访问文件的
    3. 文件对象(file object)
      1. 记录的是文件描述符，索引号
      2. 不对应真正的文件，文件open后会创建出文件对象。
      3. 文件没有close，则内核中的文件对象就没有释放
    4. 目录对象(dentry object)

软硬连接（必考）

  1. Hard link
    1. 不同的文件名对应同一个inode
    2. 不能跨越文件系统
    3. 对应系统调用link
  2. Symbolic link
    1. 存储被链接文件的文件名(而不是inode)实现链接
    2. 可跨越文件系统
    3. 对应系统调用symlink

模块依赖

  1. 一个模块A引用另一个模块B所导出的符号，我们就说模块B被模块A引用。
  2. 如果要装载模块A，必须先要装载模块B。否则，模块B所导出的那些符号的引用就不可能被链接到模块A中。这种模块间的相互关系就叫做模块依赖。

模块命令

  1. insmod <module.ko> [module parameters]

    1. Load the module
    2. 注意，只有超级用户才能使用这个命令

  2. rmmod

    1. Unload the module

  3. lsmod

    1. List all modules loaded into the kernel
    2. 这个命令和cat /proc/modules等价

  4. modprobe [-r] <module name>：这个文件是个假文件，如果打印出来，得到的是当前内核加载的模块列表，不可修改、不可删除。

  5. Load the module specified and modules it depends
