资源标识符

  • URI(统一资源标识符)
    • 用于标识某一互联网资源名称的字符串，该种标识允许用户对任何（包括本地和互联网）的资源通过特定的协议进行交互操作
    • URI是URL和URN的超集
  • URL(统一资源定位符)
    • URL是一种URI，它标识一个互联网资源，并指定对其进行操作或获取该资源的方
      法
    • 形式：
      • 协议+主机+端口+路径+查询字段+片段标识符
      • 例：http:（协议） //host.company.com（主机） :80（端口） /a/b/c.html?user=Alice&year=2018（查询） #p2（片段）
        • 查询字段
          • 查询字段是键值对 例：key1?value1&key2?value2
          • 被动参数（对显示的内容没有影响）
          • 活动参数（会影响显示的内容）
      • 超文本
        • 种类：
          • 绝对URL超链接
          • 相对URL超链接
          • 同一网页内超链接（例如目录和书签）
        • 使用
        • 编码（不含空格，用+或者%20替代空格，只能用ASCII格式）
    • 协议：http，https，file，mailto（发送邮件）
  • URN（统一资源名称）
    • 定义某事物的身份，只命名，不指定地址

HTML基础

1.HTML概述：

  • Hypertext Markup Language (HTML) ，全称为超文本标记语言，是一种标记语言。它包括一系列标签．通过这些标签可以将网络上的文档格式统一，使分散的Internet资源连接为一个逻辑整体。
  • HTML（超文本标记语言——HyperText Markup Language）是构成 Web 世界的一砖一瓦。它定义了网⻚内容的含义和结构。除 HTML 以外的其它技术则通常用来描述一个网⻚的表现与展示效果
  • 版本
    • 1989: 伯纳斯-李写了一份备忘录，提议建立一个基于互联网的超文本系统
    • 1995: RFC 1866，HTML 2成为官方
      标准语言

2.HTML 和XHTML

  • HTML最初是一种应用程序标准通用标记语言(SGML)
    • SGML是一种非常灵活的标记语言

  • XHTML:
    • XML的应用程序，SGML的一个更严格的子集

4.6应用缓存

  • 可通过cache manifest创建web应用的离线版本，引入应用程序缓存
    • 使用缓存可以：
      • 离线浏览
      • 更快
      • 减少服务器负载
  • manifest是文本文件，其可以告知浏览器被缓存的内容，其可被分为三部分：
    • CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存
    • NETWORK - 在此标题下列出的文件需要与服务器的连接，且不会被缓存
    • FALLBACK - 在此标题下列出的文件规定当⻚面无法访问时的回退⻚面（比如404 ⻚面）
  • 更新缓存：一但用户清空缓存，manifest文件被修改、程序更新，那么缓存也会进行更新。

4.7Web SQL数据库

  • 核心方法三个
    • openDatabase：这个方法使用现有的数据库或者新建的数据库创建一个数据库对象
    • transaction：这个方法让我们能够控制一个事务，以及基于这种情况执行提交或者回滚。
    • executeSql：这个方法用于执行实际的 SQL 查询。

4.8Web Workers

  • 当在 HTML ⻚面中执行脚本时，⻚面的状态是不可响应的，直到脚本已完成。（卡了）
  • web worker 是运行在后台的 JavaScript，独立于其他脚本，不会影响⻚面的性能。可以继续做任何愿意做的事情：点击、选取内容等等，而此时 web worker 在后台运行。

JS基础

客户端基础

  • HTML适合于开发静态⻚面
  • 客户端编程
    • 单独语言编写（与后端不同）
    • 程序可嵌入HTML
    • 浏览器在加载⻚面时执行程序，将程序的动态输出与HTML的静态内容集成在一起
    • 还允许用户(客户端)输入信息并对其进行处理，可在将输入提交到远程服务器之前对其进行验证

客户端脚本的好处：

  • 可用性：修改UI不返回服务器（UI加载快）
  • 效率高：可以在不等待服务器的情况下对⻚面进行小而快速的更改
  • 事件驱动： 可响应用户操作
  • 平台独立：只要浏览器支持脚本，就可以解释代码

服务器端编程的好处：

  • 安全（私有数据）
  • 兼容性：无浏览器兼容性影响
  • 功能全：包括读写文件，打开/创建链接，链接数据库…….

脚本任务

  • 向Web添加动态特征（数据表单验证，图像处理，处理cookie等）
  • 定义Web接口
  • 限制
    • 对外界可见
    • 由于安全原因，能干的事情有限
    • 功能不全，不适合大型项目开发

    作用域链

    • chatGpt 卡密！

      • avaScript的作用域链是指变量和函数在代码中被解析时可以访问的范围，其本质上是由一系列嵌套的作用域构成的链表。当JavaScript执行函数时，会首先查找当前作用域中是否存在所需变量或函数，如果找不到就会继续沿着作用域链往上查找，直到找到该变量或函数或者查找到全局作用域为止。

        作用域链的构成是由函数的嵌套关系所决定的。当函数被创建时，它会保存一个对其父级作用域的引用，这个引用就构成了作用域链的一部分。在执行函数时，会根据作用域链来查找变量和函数。

        需要注意的是，JavaScript的作用域链是静态的，即在函数定义的时候就已经确定了，不会随着函数的调用而改变。此外，当函数执行完毕后，其作用域链也会被销毁，其中的变量和函数也会被释放。

      • 在 JavaScript 中，函数内部形成了一个作用域链，它包含了函数自身的作用域以及所有父级作用域，直到全局作用域。当函数返回时，如果返回的是一个内部函数，而这个内部函数引用了外部函数的变量，那么这个内部函数就形成了一个闭包，它会保留对外部函数作用域的引用，即使外部函数已经执行完毕，这个闭包依然可以访问外部函数的变量。

        这是因为闭包中的变量实际上是被存储在堆内存中的一个对象中，而不是在栈内存中，因此即使父函数的执行上下文已经被销毁，这个堆内存中的对象仍然存在，并且可以被内部函数访问和修改。

        需要注意的是，由于闭包会持有对外部函数作用域的引用，因此如果闭包中的变量被引用了很多次，那么这些变量在内存中就会一直存在，直到闭包被释放为止，因此过度使用闭包可能会导致内存泄漏问题

名词解释

IP：internet protocol：根据端到端的设计原则，IP只为主机提供一种无连接、不可靠的、尽力而为的数据包传输服务。

TCP 传输控制协议 ：可靠，面向连接，基于字节流的传输层通讯协议

UDP 用户数据报协议 ：传输层协议，简单的面向数据报的协议，不可靠

RR 资源记录：分布式数据库的记录

Uniform Resource （Identify/Locator/Name）

URI：统一资源标识符，用于表示位于互联网上的资源

URL：统一资源定位符，用于定位互联网上的资源

URN：统一资源名称，资源的唯一名称

SEO 搜索引擎优化:决定网页在搜索时的权重

DNS 域名系统

在DNS之前，用一个文件（hosts.txt）记录域名和IP的转换，通过FTP从中央服务器下载

客户向域名服务器发出请求——本地查询——有则返回无则问根服务器——返回下一级的IP

HTTP 超文本传输协议

DOM Document object model ：提供事件类型

BOM BRowser Object Modle:赋予js与浏览器交互的能力

CORS 跨源资源共享：其他域名的网站可以访问本站资源

W3C 万维网联盟：指定Web标准的组织

XML 可扩展标记语言：允许用户自己定义自己的标记的源语言

ES2015 ECMAScript ： 一种脚本语言规格，ECMA-262

BFC 块格式化上下文：一种布局，不受环境影响

  9. 什么是Cookie和session，有什么区别？

    • Cookie和session都是用于在Web应用程序中跟踪用户状态和信息的机制，但它们实现方式和用途有所不同
    • Cookie通常用于在用户浏览器上存储少量的数据，而Session则用于在服务器上存储更大量的数据，例如用户登录状态、用户购物车等

  10. 什么是面向事件编程？什么是观察者模式？有什么特点？

    • 事件驱动编程是一种编程范式，其中程序流由事件决定，诸如用户操作(鼠标点击、按键)、传感器输出或来自其他程序/线程的消息
    • 观察者模式是一种软件设计模式，在这种模式中，一个称为主题的对象维护一个名为观察者的依赖项列表，通常通过调用它们的一个方法自动通知其任何状态变化
    • 优点：
      • 事件与观察者抽象耦合
      • 支持广播
    • 缺点：
      • 占用内存，会导致内存泄漏（失效侦听器）

  4. 同步I/O与异步I/O

    同步式 I/O(阻塞式) 异步式 I/O(非阻塞式)
    利用多线程提供吞吐量 单线程即可实现高吞吐量
    通过事件片分割和线程调度利用多核CPU 通过功能划分利用多核CPU
    需要由操作系统调度多线程使用多核 CPU 可以将单进程绑定到单核 CPU
    难以充分利用 CPU 资源 可以充分利用 CPU 资源
    内存轨迹大 ，数据局部性弱 内存轨迹小，数据局部性强
    符合线性的编程思维 不符合传统编程思维
