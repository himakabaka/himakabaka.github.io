资源标识符

  • URI(统一资源标识符)
    • 用于标识某一互联网资源名称的字符串，该种标识允许用户对任何（包括本地和互联网）的资源通过特定的协议进行交互操作
    • URI是URL和URN的超集
  • URL(统一资源定位符)
    • URL是一种URI，它标识一个互联网资源，并指定对其进行操作或获取该资源的方
      法
    • 形式：
      • 协议+主机+端口+路径+查询字段+片段标识符
      • 例：http:（协议） //host.company.com（主机） :80（端口） /a/b/c.html?user=Alice&year=2018（查询） #p2（片段）
        • 查询字段
          • 查询字段是键值对 例：key1?value1&key2?value2
          • 被动参数（对显示的内容没有影响）
          • 活动参数（会影响显示的内容）
      • 超文本
        • 种类：
          • 绝对URL超链接
          • 相对URL超链接
          • 同一网页内超链接（例如目录和书签）
        • 使用
        • 编码（不含空格，用+或者%20替代空格，只能用ASCII格式）
    • 协议：http，https，file，mailto（发送邮件）
  • URN（统一资源名称）
    • 定义某事物的身份，只命名，不指定地址

HTML基础

1.HTML概述：

  • Hypertext Markup Language (HTML) ，全称为超文本标记语言，是一种标记语言。它包括一系列标签．通过这些标签可以将网络上的文档格式统一，使分散的Internet资源连接为一个逻辑整体。

  • HTML（超文本标记语言——HyperText Markup Language）是构成 Web 世界的一砖一瓦。它定义了网⻚内容的含义和结构。除 HTML 以外的其它技术则通常用来描述一个网⻚的表现与展示效果

  • 版本

    • 1989: 伯纳斯-李写了一份备忘录，提议建立一个基于互联网的超文本系统
    • 1995: RFC 1866，HTML 2成为官方
      标准语言

2.HTML 和XHTML

  • HTML最初是一种应用程序标准通用标记语言(SGML)
    • SGML是一种非常灵活的标记语言
    • 需要一个相对复杂、宽松且通常自定义的解析器
  • XHTML:
    • XML的应用程序，SGML的一个更严格的子集
    • 真正的XHTML文档允许使用标准XML工具执行自动化处理

HTML5基础

1.神魔是HTML5

​ HTML 的最新稳定版本，HTML5 将 HTML 从用于构造一个文档的一个简单标记，扩展到一个完整的应用程序开发平台。除其他功能外，HTML5 还包括新元素和用于增强存储、多媒体和硬件访问的JavaScript APIs。

    • 赛尔号

4.5 WEB存储

  • cookie用于早期，但Web存储更牛
  • 存储两个对象
    • localStorage - 用于⻓久保存整个网站的数据，保存的数据没有过期时间，直到手动去除
    • sessionStorage - 用于临时保存同一窗口(或标签⻚)的数据，在关闭窗口或标签⻚之后将会删除这些数据。

4.6应用缓存

  • 可通过cache manifest创建web应用的离线版本，引入应用程序缓存
    • 使用缓存可以：
      • 离线浏览
      • 更快
      • 减少服务器负载
  • manifest是文本文件，其可以告知浏览器被缓存的内容，其可被分为三部分：
    • CACHE MANIFEST - 在此标题下列出的文件将在首次下载后进行缓存
    • NETWORK - 在此标题下列出的文件需要与服务器的连接，且不会被缓存
    • FALLBACK - 在此标题下列出的文件规定当⻚面无法访问时的回退⻚面（比如404 ⻚面）
  • 更新缓存：一但用户清空缓存，manifest文件被修改、程序更新，那么缓存也会进行更新。

4.7Web SQL数据库

  • H5提供了一组规范，引入了使用SQL的APIs。
  • 核心方法三个
    • openDatabase：这个方法使用现有的数据库或者新建的数据库创建一个数据库对象
    • transaction：这个方法让我们能够控制一个事务，以及基于这种情况执行提交或者回滚。
    • executeSql：这个方法用于执行实际的 SQL 查询。

4.8Web Workers

  • 当在 HTML ⻚面中执行脚本时，⻚面的状态是不可响应的，直到脚本已完成。（卡了）
  • web worker 是运行在后台的 JavaScript，独立于其他脚本，不会影响⻚面的性能。可以继续做任何愿意做的事情：点击、选取内容等等，而此时 web worker 在后台运行。

3.层叠排序

3.1层叠样式表

​ 用来解决样式说明冲突的整个排序过程称为层叠。

​ 包括：

  • 浏览器缺省格式

  • 内联格式

    直接在style中添加CSS，最弱

  • 嵌入样式表

    弱哎

  • 外部样式表

    • 链接方式：指的是使用 HTML 头部的 标签引入外部的 CSS 文件。

    • 导入方式：使用CSS规则引入外部的CSS

    • 比较

      • link 属于 HTML，通过 标签中的 href 属性来引入外部文件，而 @import 属于 CSS，所以导入语句应写在 CSS 中，要注意的是导入语句应写在样式表的开头，否则无法正确导入外部文件；
      • @import 是 CSS2.1 才出现的概念，所以如果浏览器版本较低，无法正确导入外部样式文件；
      • 当 HTML 文件被加载时，link 引用的文件会同时被加载，而 @import 引用的文件则会等⻚面全部
        下载完毕再被加载；
      • 使用DOM控制样式有差别

  • CSS如何使用

    • Web开发人员很少使用内联CSS（麻烦死了）
    • 嵌入和外部样式更受欢迎，一劳永逸，且可以一条规则设置多个元素样式。
    • 外部样式表最好，因为允许将所有样式信息保存在一个单独的文件中。然后，可以修改网站的样式，它将更新网站中的所有页面
